# sf-week design history

These three documents capture the thinking behind sf-week. They were generated by gstack skills (`/office-hours` → `/plan-ceo-review` → `/plan-eng-review`) and copied into the repo so they travel with the code.

Read in order:

1. **[01-design-doc.md](01-design-doc.md)** — Problem statement, premises, three approaches considered (keyword filter / LLM-graded / curator-with-voice), recommended approach, architecture, taste profile shape, success criteria. Generated by `/office-hours` and updated by subsequent reviews.
2. **[02-ceo-plan.md](02-ceo-plan.md)** — Scope decisions from `/plan-ceo-review` (SELECTIVE EXPANSION mode). Why the `core/` + `users/pyay/` + `sources/sf/` layout, why `/scrape` + `/skillify` for parsers, why `--replay` mode, what was deferred to v2.
3. **[03-test-plan.md](03-test-plan.md)** — Test inventory, edge cases, critical paths. Source of truth for what `/qa` and `/qa-only` should verify.

These are snapshots in time. The code in this repo is the authoritative source for what was actually built. If the docs and code disagree, the code wins — but please update the docs in the same PR.
