## Affected Pages/Routes

This is a CLI + cron tool. No web UI to test. The "interface" is:
- The Sunday email (Resend HTML render)
- The .ics calendar feed (`https://pyayaungsan.github.io/sf-week/calendar.ics`)
- The CLI: `bun run digest [--user pyay] [--replay YYYY-MM-DD] [--dry-run]`

## Key Interactions to Verify

- **CLI dry-run mode** — `bun run digest --dry-run` prints email HTML to stdout, does not send, does not commit `events.json`
- **CLI replay mode** — `bun run digest --replay 2026-05-04` loads `snapshots/2026-05-04/events.json`, runs grader + email gen, prints HTML to stdout
- **Cron flow (manually triggered via `gh workflow run`)** — runs full pipeline, sends email, commits `events.json` + `docs/calendar.ics` back to main
- **Subscribe to .ics** — Apple Calendar / Google Calendar both subscribe to the public URL, picks appear as tentative events

## Edge Cases

- **Empty events array** — grader returns `[]`, email shows "quiet week" message
- **All sources fail** — digest emits "DIGEST DEGRADED — no events fetched" footer
- **LLM API down (after retries)** — degraded mode email with raw event list, sorted by source
- **Event with ambiguous time** — fallback to canonical time per vertical (theatre 7:30pm, tech 6:30pm, art 6pm), DESCRIPTION notes time was estimated
- **Cancelled event** — appears in `.ics` with `STATUS:CANCELLED`, calendar app removes it
- **Same event from multiple sources** (Eventbrite + Luma + venue page) — fuzzy dedupe via `normalized_key` collapses to one entry
- **Event title 200+ chars** — truncated in email subject line, full title in body
- **Source returns malformed JSON** — try/catch logs to `failed-sources.log`, footer shows "⚠️ couldn't fetch: <source>"
- **Repo push collision** (rare: manual commit during cron run) — `git pull --rebase` resolves; if conflict, cron logs the failure

## Critical Paths

End-to-end flows that MUST work:

1. **Sunday cron success path:**
   scrape all sources → dedupe → grade via LLM → pick 7 + 1-3 block_now → render email + .ics → send email + commit artifacts → next Sunday at same time

2. **Replay loop:**
   edit `users/pyay/taste.md` → `bun run digest --replay 2026-04-26 --dry-run` → see new picks against old events → iterate until satisfied

3. **Calendar subscription:**
   open Apple Calendar → File → New Calendar Subscription → paste URL → events appear → cancellation propagates within one weekly cycle

4. **Source breakage detection:**
   one source's HTML changes → skillify-generated test fails on next run → digest footer shows "⚠️ couldn't fetch: <source>" → fix the parser → next run includes that source again
